# Roadmap: Service Core

Обновлено: 2026-03-23

## Правило фокуса
- Главный файл статуса: `CHAT_HANDOFF.md`
- Этот файл описывает только направление и этапы.
- Детальные "где остановились" всегда фиксируются в `CHAT_HANDOFF.md`.

## Текущий фокус (этап)
Стабилизация core-flow сервиса:
1. Регистрация
2. Логин
3. Создание магазина
4. Управление товарами
5. Проверка Telegram WebApp

## Этап 1: Core-flow Hardening
- Привести UX цепочки к единообразному виду (стили, ошибки, loading-state).
- Убрать критичные падения сценариев в ручном и e2e режиме.
- Закрыть пробелы в валидации обязательных полей.

## Этап 2: Настройки магазина (операционный минимум)
- Единая форма настроек магазина (название, доставка, chat/webhook поля).
- Понятные статусы сохранения и ошибок.
- Проверка ownership/permission на API и UI.

## Этап 3: Интеграционные блоки MVP
- Bot token validation (`getMe`).
- Checkout + Order model + базовый order flow.
- Подготовка к платежам и уведомлениям.

## Done Definition для текущего этапа
- Core-flow проходит вручную без блокеров.
- Smoke e2e сценарии воспроизводимы.
- Все остановки/блокеры отражены в `CHAT_HANDOFF.md`.
