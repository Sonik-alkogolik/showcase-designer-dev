# QA Release Checklist

Обновлено: 2026-04-19

## Как использовать

1. Перед релизом пройти все блоки сверху вниз.
2. Для каждого теста отметить:
- `PASS` — успешно
- `FAIL` — найден баг (ссылка на issue)
- `N/A` — не применимо к релизу
3. Релиз только если нет критичных `FAIL`.

---

## 1) Auth и доступ

- [ ] Регистрация нового пользователя
- [ ] Логин по email/паролю
- [ ] Логаут
- [ ] Авто-редирект на `/login` при протухшей сессии
- [ ] Пользователь не видит чужие магазины (проверка `own.shop`)
- [ ] Смена пароля после временного пароля (forced change flow)

## 2) Восстановление пароля

- [ ] Email reset: отправка письма (`/forgot-password`)
- [ ] Email reset: установка нового пароля (`/reset-password`)
- [ ] Telegram fallback: отправка кода
- [ ] Telegram fallback: код работает только 1 раз
- [ ] Telegram fallback: просроченный код отклоняется
- [ ] После Telegram reset пользователь обязан сменить пароль

## 3) Магазин и настройки

- [ ] Создание магазина
- [ ] Редактирование настроек магазина
- [ ] Проверка токена бота и статуса подключения
- [ ] Витрина открывается: `/app?shop={id}`

## 4) Каталог и импорт XLSX

- [ ] Создание/редактирование товара вручную
- [ ] Привязка товара к нескольким категориям
- [ ] Импорт XLSX (валидный файл)
- [ ] Импорт XLSX (битые строки) — валидные загружаются, ошибки считаются
- [ ] Лимиты тарифа соблюдаются при импорте
- [ ] Для относительных URL изображений работает base URL
- [ ] История импортов и статус async-импорта

## 5) Заказы и Telegram Mini App

- [ ] Создание заказа из mini-app
- [ ] Заказ появляется в dashboard
- [ ] Обновление статуса заказа
- [ ] Уведомление менеджеру (если настроен chat id)
- [ ] UI mini-app: карточки, категории, поиск, footer не перекрывает контент

## 6) Dashboard UX

- [ ] Sidebar корректно отображается на desktop
- [ ] Sidebar корректно отображается на mobile
- [ ] Раздел Help не содержит внутренних служебных ссылок
- [ ] Язык панели: переключатель сохраняет выбор (если i18n подключен)

## 7) MoonShine / Admin

- [ ] Пользователи: поля отображаются корректно
- [ ] Подписки: в edit нет dropdown пользователя
- [ ] Магазины/Категории/Заказы отображаются корректно
- [ ] Удаление пользователя каскадно очищает связанные данные

## 8) Нагрузочный smoke (минимум)

- [ ] 20 параллельных запросов к публичной витрине без 500
- [ ] 20 параллельных запросов к `/api/profile` под авторизацией без 500
- [ ] Очередь воркера активна (`supervisorctl status`)

## 9) Логи и диагностика

- [ ] `storage/logs/laravel.log` без новых критичных ошибок после прогона
- [ ] Нет повторяющихся ошибок 500 на ключевых endpoints
- [ ] Очередь не копит зависшие jobs

## 10) Деплой-проверка

- [ ] `git pull origin main`
- [ ] `php artisan migrate --force`
- [ ] `npm run build`
- [ ] Копирование `client/dist` в `public`
- [ ] `php artisan optimize:clear`
- [ ] Smoke после деплоя: login + profile + shops + mini-app

---

## Что автоматизировать в первую очередь

1. API feature tests (Laravel):
- auth/login/logout
- password reset email + telegram fallback
- own.shop доступы
- import limits

2. E2E (Playwright):
- login -> dashboard -> shops/products
- forgot password (email/telegram mock)
- mini-app smoke (`/app?shop=...`)

3. Contract checks:
- ключевые API-ответы (`/api/profile`, `/api/shops`, `/api/subscription/plans`)

4. CI-гейт:
- `phpunit` + e2e smoke + сборка фронта
