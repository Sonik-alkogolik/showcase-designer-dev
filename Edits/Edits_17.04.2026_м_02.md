# План фронтенд-исправлений WebApp (Telegram)

Обновлено: 2026-04-17
Файл правок: `Edits_17.04.2026_м_02.md`

## Цель

Стабилизировать и привести к целевому UX фронтенд Telegram WebApp проекта `t-go`:

1. заменить `<title>Vite App</title>` на бренд проекта;
2. восстановить сломанную вёрстку каталога/профиля;
3. зафиксировать предсказуемую структуру верхней зоны, контента и нижней навигации;
4. убрать наезды контента на футер;
5. сделать карточки товаров удобными на мобильном экране Telegram.

---

## Факты по текущему состоянию (по коду и скриншотам)

1. В `client/index.html` сейчас стоит `<title>Vite App</title>`.
2. В шапке каталога используется `WEB APP STORE`.
3. Категории сейчас отображаются как горизонтальные chips, а нужно выпадающее меню.
4. Карточки и текст местами ломают поток, наезжают друг на друга.
5. В профиле элементы выглядят как «сырой HTML» (кнопки/аватар/отступы без единого стиля).
6. Нижняя навигация визуально и по слоям конфликтует с контентом.

---

## Целевое поведение (что должно быть после правок)

## 1) Базовый бренд

- `<title>` в `client/index.html` = `t-go`.
- Верхний kicker в каталоге: вместо `WEB APP STORE` использовать `t-go shop`.

## 2) Единый контейнер

Для основного контента WebApp:
- `width: 98%`
- `margin: 0 auto`
- единые горизонтальные отступы
- контент не прилипает к краям Telegram viewport

## 3) Верхняя зона каталога (одна строка)

В одну строку:
- текст `t-go shop`
- название магазина
- иконка поиска

Ниже этой строки:
- выпадающий список категорий
- слайдер товаров

## 4) Поиск в fullscreen-режиме

При клике на иконку поиска:
- открывается overlay в верхней части экрана на всю ширину;
- вверху input для поиска;
- ниже отображается список найденных товаров;
- закрытие поиска: кнопка `X` и/или клик вне зоны.

## 5) Категории как dropdown

Вместо chips:
- селект/выпадающий список;
- дефолт: `Весь магазин`;
- при выборе категории показываются соответствующие товары.

## 6) Карточки товаров

- карточка на всю ширину родительского контейнера (`1 колонка`);
- изображение/заголовок/цена/кнопка действия выровнены стабильно;
- описание в свёрнутом виде (ограниченная высота, например line-clamp);
- по клику на стрелку/кнопку раскрытия описание разворачивается полностью (`height: auto` / expanded state);
- добавить кнопку «Показать ещё»:
  - первичный рендер: 10 товаров;
  - по клику догрузка следующей порции (например +10).

## 7) Футер/нижняя навигация

- футер всегда видимый и поверх контента;
- контент не заезжает под футер;
- добавить нижний safe padding у контентной области (по высоте футера + отступ);
- для футера фиксированное позиционирование и высокий слой (`z-index`).

## Архитектурный план реализации

## Этап 1. Брендинг и быстрые правки

Файлы:
- `client/index.html`
- `client/src/views/telegram/WebAppView.vue`

Действия:
1. `title` -> `t-go`.
2. `WEB APP STORE` -> `t-go shop`.

Критерий готовности:
- в браузере и Telegram WebView отображается новый title/заголовок.

## Этап 2. Рефактор шапки каталога

Файл:
- `client/src/views/telegram/WebAppView.vue`

Действия:
1. Перестроить header в single-row layout.
2. Вынести поиск в overlay-блок.
3. Привязать открытие/закрытие поиска к явному состоянию.

Критерий готовности:
- верхняя зона не ломается на ширинах 320-430px.

## Этап 3. Категории и слайдер

Файл:
- `client/src/views/telegram/WebAppView.vue`

Действия:
1. Chips заменить на dropdown/select.
2. Сохранить фильтрацию категорий через текущий API.
3. Проверить слайдер после изменения структуры.

Критерий готовности:
- выбор категории корректно фильтрует список без визуальных сдвигов.

## Этап 4. Карточки и «Показать ещё»

Файл:
- `client/src/views/telegram/WebAppView.vue`

Действия:
1. Перевести сетку в 1 колонку для WebApp каталога.
2. Добавить локальный pagination state:
   - `visibleCount = 10`
   - кнопка «Показать ещё».
3. Сделать collapse/expand описания у карточки.

Критерий готовности:
- первая загрузка = 10 карточек;
- контент карточки не ломает layout;
- раскрытие описания работает независимо для каждой карточки.

## Этап 5. Футер и слои

Файл:
- `client/src/views/telegram/WebAppView.vue` (стили)

Действия:
1. Зафиксировать нижний nav/footer.
2. Добавить `padding-bottom` контенту.
3. Проверить z-index, чтобы элементы списка/модалки не перекрывали футер некорректно.

Критерий готовности:
- нет налезания текста/кнопок на футер на длинных списках.

## Этап 6. Профиль (визуальная стабилизация)

Файл:
- `client/src/views/telegram/WebAppView.vue`

Действия:
1. Нормализовать карточку профиля (аватар, username, имя).
2. Привести кнопки к единому стилю WebApp.
3. Убрать «сырой» вид стандартных кнопок.

Критерий готовности:
- профиль визуально соответствует общему стилю и не разваливается.

---

## Технические требования по CSS

1. Все размеры и отступы проверять на мобильной ширине (Telegram in-app browser).
2. Использовать единые CSS-переменные для spacing/z-index.
3. Избегать жёстких абсолютных высот без необходимости.
4. Для длинного текста:
- `word-break` / `overflow-wrap`;
- ограничение в свёрнутом режиме;
- плавное раскрытие по классу expanded.

## Риски и контроль

1. Риск: текущие стили WebApp завязаны на старую структуру DOM.
- Контроль: рефактор по этапам с проверкой после каждого шага.

2. Риск: Telegram WebView отличается от обычного браузера.
- Контроль: финальная проверка именно в Telegram.

3. Риск: поиск fullscreen может конфликтовать с fixed footer.
- Контроль: отдельный z-index stack и тест перекрытий.
