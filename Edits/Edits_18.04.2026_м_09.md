# Этап 8: каскадное удаление пользователя из MoonShine

Обновлено: 2026-04-18
Файл правок: `Edits_18.04.2026_м_09.md`

## Задача

При удалении пользователя через MoonShine нужно удалять все данные, связанные с этим пользователем:
- магазины;
- товары;
- категории;
- заказы;
- запуски импорта;
- подписки.

## Что реализовано

1. Добавлена защитная очистка в `User` model (`deleting` hook):
- удаляются `importRuns`, `subscriptions`, `sanctum tokens`;
- удаляются все магазины пользователя (через model delete).

2. Добавлена доменная очистка в `Shop` model (`deleting` hook):
- удаляются `importRuns`, `orders`;
- удаляются все товары магазина;
- удаляются все категории магазина.

3. Добавлена очистка локальных картинок в `Product` model (`deleting` hook):
- если `image` хранится как `/storage/...`, файл удаляется с диска `public`.

## Измененные файлы

1. `app/Models/User.php`
2. `app/Models/Shop.php`
3. `app/Models/Product.php`

## Итог

Удаление пользователя через админку теперь гарантированно чистит все связанные данные на уровне приложения, даже если где-то есть расхождение в FK-каскадах.
