# Этап 3: фронт для async-импорта (модалка)

Обновлено: 2026-04-18
Файл правок: `Edits_18.04.2026_м_04.md`

## Что сделано

1. Модалка импорта переведена на async API:
- запуск через `POST /api/shops/{shop}/import/async`;
- получение `import_run_id`;
- автополлинг статуса через `GET /api/shops/{shop}/import/status/{run}`.

2. Добавлена история последних импортов:
- загрузка через `GET /api/shops/{shop}/import/history`;
- вывод последних запусков со статусами.

3. Добавлен fallback:
- если async endpoint недоступен (`404/405`), модалка откатывается на старый sync endpoint `POST /api/shops/{shop}/import`.

4. Улучшено UX результата:
- отображение статуса run (в очереди/обрабатывается/завершен/ошибка);
- ручная кнопка обновления статуса;
- автоостановка поллинга при завершении.
