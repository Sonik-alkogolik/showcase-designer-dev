# Discussion — AI-to-AI Коммуникация

Эта папка содержит задачи в формате **структурированного диалога между AI агентами**.

---

## 🚀 Быстрый старт

| Шаг | Команда | Описание |
|-----|---------|----------|
| **1** | `@Qwen @Codex СТАРТ` | Сканирование проекта + запуск диалога |
| **2** | `@Qwen @Codex ЗАВЕРШЕНИЕ` | Фиксация результатов + обновление статусов |

**Подробные инструкции:**
- [start.md](./start.md) — как начать сессию
- [end.md](./end.md) — как завершить сессию

## 🤖 Роли в обсуждении

| Роль | AI | Обязанности |
|------|-----|-------------|
| **Постановщик** | 🤖 AI Qwen | Анализ задачи, декомпозиция, план, критерии приёмки |
| **Исполнитель** | 🤖 AI Codex | Реализация, технический анализ, вопросы, отчёт |
| **Человек** | 👤 Разработчик | Утверждение стратегии, решения, вопросы к AI |

## 📁 Структура файла задачи

```
┌─────────────────────────────────────────────────────────┐
│  # Discussion #N: Название                              │
│  │                                                      │
│  ├── 📋 Метаданные (статус, приоритет, роли)           │
│  │                                                      │
│  ├── 📋 План задачи (поэтапно с чекбоксами)            │
│  │   ├── Этап 1: [цель, команды, ожидаемый результат] │
│  │   ├── Этап 2: ...                                   │
│  │   └── ...                                           │
│  │                                                      │
│  ├── 💬 Обсуждение задачи                              │
│  │   ├── 🤖 AI Qwen (Постановщик) — рассуждения       │
│  │   ├── 🤖 AI Codex (Исполнитель) — анализ/вопросы   │
│  │   ├── 🤖 AI Qwen — ответы                           │
│  │   └── 👤 Человек — решения                          │
│  │                                                      │
│  └── 📎 Приложения (файлы, ссылки)                     │
└─────────────────────────────────────────────────────────┘
```

## 🔄 Рабочий процесс

```
1️⃣ AI Qwen создаёт файл задачи
   │
   ▼
2️⃣ AI Codex читает план, пишет анализ и вопросы
   │
   ▼
3️⃣ AI Qwen отвечает на вопросы, корректирует план
   │
   ▼
4️⃣ Человек утверждает стратегию
   │
   ▼
5️⃣ AI Codex реализует, заполняет статусы этапов
   │
   ▼
6️⃣ Человек принимает работу, обновляет CHAT_HANDOFF.md
```

## 📝 Как заполнять секции

### Для AI Qwen (Постановщик)

```markdown
### 🤖 AI Qwen (Постановщик) — 2026-03-30

**Рассуждения по постановке:**

[почему задача важна, контекст]

**Почему выбран такой план:**

[обоснование структуры плана]

**Риски:**

[что может пойти не так]

**Вопрос к AI Codex:**

[технический вопрос на уточнение]
```

---

### Для AI Codex (Исполнитель)

```markdown
### 🤖 AI Codex (Исполнитель) — 2026-03-30

**Рассуждения по реализации:**

[анализ плана, технические нюансы]

**Анализ плана:**

- Этап 1: [комментарии]
- Этап 2: [комментарии]

**Вопросы к AI Qwen:**

- [вопрос 1]
- [вопрос 2]

**Предложения:**

[как можно оптимизировать]
```

---

### Для Человека (Разработчик)

```markdown
### 👤 Человек — 2026-03-30

**Решения:**

- Выбранный вариант: [A / B / C / D]
- Комментарий: [текст]

**Вопросы к AI:**

[что нужно уточнить]
```
